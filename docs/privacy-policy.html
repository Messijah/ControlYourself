1. Introduction

This Privacy Policy describes how Urge ("Application," "we," "our," or "us") collects, uses, and discloses information about users of the Urge mobile application.

Urge is designed with privacy as a fundamental principle. The Application operates entirely on-device and does not transmit user data to external servers or third parties, except as specifically described in this Privacy Policy.

2. Information Collection and Storage

2.1 Data Collection Practices

Urge does not collect, transmit, or process any personally identifiable information, usage analytics, or behavioral data from users on external servers.

2.2 Local Data Storage

The Application stores the following information exclusively on the user's device using iOS local storage mechanisms (UserDefaults and App Groups):

  • User configuration preferences and application settings
  • Timer and countdown state information
  • Usage statistics and progress tracking data
  • Subscription status information (managed through Apple's infrastructure)

2.3 Data Retention and Deletion

All locally stored data:

  • Remains exclusively within the device's secure storage environment
  • Is subject to iOS security and encryption protocols
  • Is permanently deleted upon application uninstallation
  • Can be manually reset through the Application's settings interface

3. Third-Party Services

3.1 Apple Services

Urge integrates exclusively with Apple-provided services:

StoreKit (In-App Purchases)

Used solely for processing subscription transactions. Apple Inc. handles all payment information in accordance with their privacy policy. We receive only subscription status information (active/inactive) and do not have access to financial or payment details.

3.2 Services Not Used

The Application explicitly does not integrate with or utilize:

  • Analytics platforms or services (including but not limited to Google Analytics, Firebase Analytics)
  • Crash reporting or diagnostic services
  • Advertising networks or tracking software development kits (SDKs)
  • Social media platforms or integration services
  • Cloud storage or synchronization services
  • Any third-party data collection, processing, or transmission tools

4. Notifications and Local Features

4.1 Local Notifications

When enabled by the user, notifications are generated locally on the device using iOS notification services. No server-side communication or data transmission occurs in connection with notification delivery.

4.2 Dynamic Island and Live Activities

Timer displays utilize iOS ActivityKit framework and operate entirely on-device. No data is transmitted to external servers in connection with these features.

5. Data Security

The Application's security architecture relies on iOS-provided security mechanisms:

  • Data is protected by device-level encryption and security features
  • No network transmission of user data eliminates interception and man-in-the-middle attack vectors
  • Access to locally stored data is controlled by iOS security mechanisms and user device authentication (passcode/biometric)

6. Children's Privacy

Urge does not knowingly collect information from individuals of any age. The Application's privacy-by-design architecture ensures compliance with:

  • Children's Online Privacy Protection Act (COPPA)
  • General Data Protection Regulation (GDPR) requirements for processing data of minors
  • Other applicable children's privacy regulations

7. Rights Under GDPR

For users in the European Economic Area (EEA), United Kingdom, and Switzerland, the following rights apply under the General Data Protection Regulation:

7.1 Right of Access

All user data is accessible within the Application's interface. No data is held by us externally.

7.2 Right to Erasure

Users may delete all locally stored data by:

  • Uninstalling the Application
  • Using the in-application reset function

7.3 Right to Data Portability

Not applicable as no personal data is collected or held by the data controller.

7.4 Right to Object

Not applicable as no data processing activities occur beyond local device storage.

8. California Privacy Rights (CCPA)

California residents have specific rights under the California Consumer Privacy Act. However, as Urge does not collect, sell, or share personal information as defined by the CCPA, most provisions do not apply.

We do not:

  • Collect personal information from California residents
  • Sell personal information
  • Share personal information for cross-context behavioral advertising purposes

9. International Data Transfers

The Application does not transfer data internationally. All data processing occurs locally on the user's device.

10. Changes to This Privacy Policy

We reserve the right to modify this Privacy Policy at any time. Material changes will be communicated through:

  • Update to this document with revised "Last Updated" date
  • In-application notification regarding policy changes
  • Request for user consent if data collection practices materially change

Continued use of the Application following policy modifications constitutes acceptance of the updated Privacy Policy.
